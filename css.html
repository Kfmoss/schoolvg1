1. Introduksjon

Når vi arbeider med CSS (Cascading Style Sheets), kan vi plassere stilreglene våre på tre forskjellige steder: inline, intern og ekstern. Denne siden oppsummerer presentasjonen og gir små kodeeksempler underveis.

💡
Best praksis er å holde HTML ren og legge mest mulig stil i en ekstern CSS-fil.

2. Grunnleggende CSS

De tre plasseringene for CSS:

  • Inline – stil direkte på et enkelt HTML‑element.
  • Intern – stil i en <style>-tagg i <head>.
  • Ekstern – stil i en separat .css-fil koblet inn med <link>.

2.1 Inline CSS

Inline CSS brukes når vi skriver stilregler direkte på et element. Dette kan bli tungvint på større sider, og det overstyrer ofte eksterne regler.

<h1 style="color: teal; margin-bottom: .5rem;">Hei verden</h1>
⚠️
Bruk sparsommelig – inline-regler er vanskelige å vedlikeholde og overstyre.

2.2 Intern CSS

Intern CSS plasseres i dokumentets <head> innenfor en <style>-tagg.

<head>
  <style>
    body { font-family: system-ui; }
    h1 { font-size: 2rem; }
  </style>
</head>

Regler her kan overstyre eksterne regler dersom selektorene er like.

2.3 Ekstern CSS

Koble en separat CSS-fil i <head> for ryddig og gjenbrukbar stil.

<head>
  <link rel="stylesheet" href="styles.css">
</head>

Hvorfor eksternt stilark?

  • Holder HTML renere og mer lesbar.
  • Enklere å gjenbruke stil på tvers av flere sider.
  • Færre konflikter enn med inline stil.

3. Class- og ID-selektorer

Vi kan målrette spesifikke elementer ved å tildele klasser og id-er i HTML. De har ulike bruksområder:

3.1 Klasse-selektor .klasse

  • Gjenbrukbar – et element kan ha flere klasser.
  • I CSS: .kort. I HTML: class="kort" (uten punktum).
/* CSS */
.kort { padding: 1rem; border-radius: .75rem; }

/* HTML */
<div class="kort">Innhold</div>

3.2 ID-selektor #id

  • Unik per side – bruk på ett element.
  • I CSS: #hero. I HTML: id="hero" (uten #).
/* CSS */
#hero { scroll-margin-top: 80px; }

/* HTML */
<section id="hero">...</section>

3.3 Selektorgruppering

Unngå repetisjon ved å gi flere selektorer samme regler ved hjelp av komma.

h1, h3 { color: rebeccapurple; }

3.4 Selektorkjeding (målrett spesifikt element)

Når du må style én bestemt lenke inne i en overskrift med en klasse:

/* Påvirker bare <a> inne i <h3 class="enklasse"> */
h3.enklasse a { text-decoration: underline; }

3.5 Type-selektor

Bruk HTML-taggen som selektor for å style alle forekomster av den typen.

p { line-height: 1.7; }

3.6 Selektor-spesifisering

Har du en klasse på mange elementer, men vil endre én bestemt <p> med klassen? Spesifiser typen + klassen:

/* Påvirker bare <p> som har klassen .info */
p.info { font-style: italic; }

3.7 Slekttre-/descendant-selektor

Målrett elementer ut fra hvor de ligger i HTML-strukturen, uten ekstra klasser eller id-er.

/* Endre farge på lenker i menyen i header */
header nav ul li a { color: cornflowerblue; }

Oppsummering

  • Foretrekk ekstern CSS for orden og gjenbruk.
  • Bruk klasser for gjenbruk, ID for unike ankre.
  • Grupper, kjed og spesifiser selektorer for presis styling – uten overflødige klasser.

Tips Lag en egen styles.css og hold inline-stil til et minimum.
