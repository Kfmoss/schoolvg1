Introduksjon

Når vi lager nettsider, er HTML grunnmuren som bygger opp innholdet – tekst, bilder, overskrifter og lenker. Men uten CSS blir nettsiden likevel ganske kjedelig og utydelig. CSS (Cascading Style Sheets) er språket vi bruker for å bestemme hvordan en nettside ser ut: farger, skrifttyper, plassering av elementer og hvordan siden oppfører seg på ulike skjermer. Å lære CSS handler derfor ikke bare om “pynt”, men om kommunikasjon og tilpasning. En god nettside skal ikke bare fungere, den skal være forståelig, brukervennlig og tilpasset den gruppen mennesker den er laget for. I dette prosjektet skal dere derfor jobbe med hvordan dere kan bruke CSS til å gjøre en idé tydeligere og enklere å oppleve.

💡
Best praksis er å holde HTML ren og legge mest mulig stil i en ekstern CSS-fil.

Grunnleggende CSS

Når vi arbeider med CSS (Cascading Style Sheets), kan vi plassere stilreglene våre på tre forskjellige steder: inline, intern og ekstern.

De tre plasseringene for CSS:

  • Inline – stil direkte på et enkelt HTML‑element.
  • Intern – stil i en <style>-tagg i <head>.
  • Ekstern – stil i en separat .css-fil koblet inn med <link>.

Inline CSS

Inline CSS brukes når vi skriver stilregler direkte på et element. Dette kan bli tungvint på større sider, og det overstyrer ofte eksterne regler.

<h1 style="color: teal; margin-bottom: .5rem;">Hei verden</h1>
⚠️
Bruk sparsommelig – inline-regler er vanskelige å vedlikeholde og overstyre.

Intern CSS

Intern CSS plasseres i dokumentets <head> innenfor en <style>-tagg.

<head>
  <style>
    body { font-family: system-ui; }
    h1 { font-size: 2rem; }
  </style>
</head>

Regler her kan overstyre eksterne regler dersom selektorene er like.

Ekstern CSS

Koble en separat CSS-fil i <head> for ryddig og gjenbrukbar stil.

<head>
  <link rel="stylesheet" href="styles.css">
</head>

Hvorfor eksternt stilark?

  • Holder HTML renere og mer lesbar.

  • Enklere å gjenbruke stil på tvers av flere sider.

  • Færre konflikter enn med inline stil.

CSS-selektorer (CSS-velgere)

En CSS-selektor er starten på en CSS-regel. Ordet selektor betyr egentlig «velger». Den brukes til å peke ut hvilke deler av en nettside (HTML-elementer) som skal få en bestemt stil. Når du lager en regel i CSS, sier selektoren altså til nettleseren: «Dette er de elementene du skal endre stilen på.»

ID-selektor

En ID-selektor i CSS brukes når du vil gi en spesiell stil til ett bestemt element på nettsiden. Hvert HTML-element kan få et eget «navneskilt» som kalles en id. Dette navneskiltet skal være unikt, altså at det ikke brukes på flere elementer samtidig.

Når du har gitt et element et slikt id-navn i HTML, kan du i CSS vise til dette navnet ved å sette et nummertegn (#) foran. På den måten forteller du nettleseren: «Jeg vil at akkurat dette elementet, med dette navnet, skal få den stilen jeg beskriver her.»

Klasse-selektor

En class-selektor i CSS bruker du når du vil gi flere elementer på en nettside samme utseende eller stil. Det er altså en gruppe-etikett du kan sette på elementer som skal se like ut. Et eksempel kan være Hvis du har mange knapper som alle skal ha samme farge og form, kan du gi dem en felles klasse. Du kan bruke samme klasse på så mange elementer du vil, noe som gjør det enklere å holde designet konsistent og ryddig.

Når du lager en klasse i CSS, må du alltid bruke punktum (.) foran navnet på klassen. Hvis du har en klasse som heter "boks", så må du skrive et punktum foran navnet når du refererer til den i CSS.

Klasse-selektor .klasse

  • Gjenbrukbar – et element kan ha flere klasser.

  • I CSS: .i1. I HTML: class="i1" (uten punktum).

/* CSS */ 
.i1{
    font-size: 2.125rem;
    color:blue;
    font-style: italic;
  }

/* HTML */
<li class="i1">en bok</li>

ID-selektor #id

  • Unik per side – bruk på ett element.

  • I CSS: #i3. I HTML: id="i3" (uten #).
/* CSS */
#i3{
    font-size: 3.45rem;
    color:chocolate;
  }  
/* HTML */
<li id="i3">...</li>

Selektorgruppering

I stedet for å skrive de samme stilene for hvert enkelt element, kan du gruppere selektorene ved å skille dem med komma. Da får alle elementene i gruppen samme utseende.

Hvis du vil at både overskrifter og avsnitt skal ha samme skriftfarge, trenger du ikke skrive to regler. Du kan samle dem i én. Dette gjør at du unngår gjentakelse, reduserer antall linjer med kode og gjør det lettere å oppdatere designet senere. Hvis du for eksempel vil endre fargen på overskriftene, trenger du bare å gjøre det på ett sted i stedet for flere. 

h1, p { color: rebeccapurple; }
#i2, .nav{ background-color: "red"}

Selektorkjeding (målrett spesifikt element)

Selektorkjeding i CSS betyr at du kombinerer flere selektorer for å målrette et mer spesifikt element. Dette brukes når du vil style bare et bestemt element i en bestemt sammenheng. 

article p {color: gray; }

Her betyr article p at bare avsnitt <p > som ligger inni et <article>-element får grå tekst. Avsnitt utenfor article blir ikke påvirket.

Når du må style én bestemt lenke inne i en overskrift med en klasse:

/* Påvirker bare <a> inne i <h3 class="enklasse"> */

h3.enklasse a { text-decoration: underline; }

Type-selektor

Type-selektoren velger elementer etter HTML-navn, som <h1>, <p>, <div>, <a> osv. Den brukes når du vil gi samme stil til alle elementer av en bestemt type.

Bruk HTML-taggen som selektor for å style alle forekomster av den typen.

p { line-height: 1.7; }

Selektor-spesifisering

Har du en klasse på mange elementer, men vil endre én bestemt <p> med klassen? Spesifiser typen + klassen:

/* Påvirker bare <p> som har klassen .info */
p.info { font-style: italic; }

Slekttre-/descendant-selektor

Målrett elementer ut fra hvor de ligger i HTML-strukturen, uten ekstra klasser eller id-er.

/* Endre farge på lenker i menyen i header */
header nav ul li a { color: cornflowerblue; }

Oppsummering

  • Foretrekk ekstern CSS for orden og gjenbruk.

  • Bruk klasser for gjenbruk, ID for unike ankre.

  • Grupper, kjed og spesifiser selektorer for presis styling – uten overflødige klasser.

Tips Lag en egen styles.css og hold inline-stil til et minimum.
